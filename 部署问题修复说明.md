# 部署问题修复说明

## 问题概述

在部署到 `https://neihou.cn/clock/` 时遇到以下错误：

1. ❌ `GET https://neihou.cn/clock/none 404 (Not Found)`
2. ❌ `GET https://neihou.cn/clock/assets/bgm/ 403 (Forbidden)`  
3. ❌ `GET https://neihou.cn/favicon.svg 404 (Not Found)`
4. ❌ 音乐列表无法加载，提示"未找到音乐文件"

## 修复方案

### ✅ 1. 修复了 Favicon 路径问题
**原因**: 使用了绝对路径 `/favicon.svg`，在子目录部署时会指向错误的位置  
**修改**: 改为相对路径 `./favicon.svg`

### ✅ 2. 解决了 BGM 403 错误
**原因**: 生产服务器禁止目录浏览，导致无法动态读取音乐文件列表  
**解决方案**: 
- 创建 `assets/bgm/music-list.json` 音乐索引文件
- 修改加载逻辑：优先读取索引文件，如果失败才尝试目录浏览
- 保持了原有的动态加载机制（不使用硬编码列表）

### ✅ 3. 创建了自动化工具
- **`scripts/generate-music-list.sh`**: 自动扫描并生成音乐索引文件
- **`deploy.sh`**: 一键部署脚本，自动化整个部署流程
- **`.htaccess`**: Apache 服务器配置文件
- **`nginx.conf`**: Nginx 服务器配置示例

## 使用方法

### 快速部署（推荐）

```bash
# 1. 生成音乐索引
./scripts/generate-music-list.sh

# 2. 一键部署
./deploy.sh user@neihou.cn /var/www/html/clock
```

### 验证部署

访问 `https://neihou.cn/clock/` 并检查：
- ✅ 页面正常显示
- ✅ Favicon 显示正确
- ✅ 音乐列表可以加载（显示 3 首歌曲）
- ✅ 音乐可以正常播放
- ✅ 浏览器控制台无 404 错误

## 音乐加载机制

应用现在按以下顺序尝试加载音乐：

1. **Capacitor 环境**（移动应用）→ 从应用文件系统读取
2. **music-list.json**（推荐）→ 从索引文件读取
3. **目录浏览**（开发环境）→ 直接解析目录
4. **显示空列表** → 提示用户添加音乐

这样既保持了灵活性，又解决了生产环境的部署问题。

## 添加新音乐

```bash
# 1. 将音乐文件复制到目录
cp ~/Music/new-song.mp3 assets/bgm/

# 2. 重新生成索引
./scripts/generate-music-list.sh

# 3. 上传到服务器
scp assets/bgm/new-song.mp3 user@neihou.cn:/var/www/html/clock/assets/bgm/
scp assets/bgm/music-list.json user@neihou.cn:/var/www/html/clock/assets/bgm/

# 4. 刷新页面即可看到新音乐
```

## 控制台输出说明

部署后，浏览器控制台可能显示：

```
未找到 music-list.json，尝试读取目录...
无法读取 bgm 目录列表: 无法访问 bgm 目录
✅ 从 music-list.json 加载了 3 首音乐
```

这是**正常的**！表示应用尝试了多种方式，最终成功从 `music-list.json` 加载。

如果看到一个 403 错误但音乐正常加载，可以忽略（这是尝试读取目录时的正常回退）。

- ✅ 支持多环境：开发/生产/移动应用
- ✅ 用户友好：普通用户也能自己添加音乐
