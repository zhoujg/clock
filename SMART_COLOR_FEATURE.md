# 智能颜色调整功能说明

## 功能概述

智能颜色调整功能可以根据背景图片或背景颜色的亮度，自动调整顶部日期、星期和天气信息的显示颜色，确保文字在任何背景下都清晰可读。

## 工作原理

### 1. 背景检测
- **图片背景**：系统会加载背景图片并分析顶部区域（日期显示区域）的平均亮度
- **纯色背景**：系统会直接分析背景颜色的亮度值

### 2. 亮度分析
- 使用感知亮度公式：`brightness = 0.299 * R + 0.587 * G + 0.114 * B`
- 采样区域：顶部中心 30%-70% 宽度，5%-20% 高度的区域
- 亮度范围：0-100%

### 3. 颜色调整策略

#### 深色背景（亮度 < 50%）
- 文字颜色：浅白色 `rgba(255, 255, 255, 0.95)`
- 文字阴影：深色 `rgba(0, 0, 0, 0.5)`
- 分隔线：浅色 `rgba(255, 255, 255, 0.6)`

#### 浅色背景（亮度 ≥ 50%）
- 文字颜色：深黑色 `rgba(0, 0, 0, 0.85)`
- 文字阴影：浅色 `rgba(255, 255, 255, 0.8)`
- 分隔线：深色 `rgba(0, 0, 0, 0.4)`

### 4. 平滑过渡
所有颜色变化都使用 0.5 秒的平滑过渡动画（`transition: 0.5s ease`）

## 影响的元素

1. **日期文字**（#dateText）
   - 颜色
   - 文字阴影

2. **星期文字**（#weekText）
   - 颜色
   - 文字阴影
   - 左侧分隔线颜色

3. **天气组件**（.weather-display）
   - 温度文字颜色和阴影
   - 天气描述文字颜色和阴影
   - 天气图标滤镜效果
   - 容器背景和边框

4. **音乐播放器**（.bgm-player-container）
   - 播放器容器背景和边框
   - 播放器标题文字
   - 当前曲目名称
   - 艺术家名称
   - 时间显示
   - 播放列表曲目文字和图标
   - 控制按钮背景和边框
   - 各个半透明容器（当前曲目、音量控制、播放器头部等）
   - 所有文字元素的透明度

## 使用方式

### 自动触发
系统会在以下情况自动触发智能颜色调整：

1. **页面加载时**：加载 500ms 后自动分析
2. **背景更改时**：通过 MutationObserver 监听 body 的 style 属性变化
3. **设置恢复时**：从本地存储恢复背景设置后

### 手动触发
如果需要手动触发颜色调整，可以在浏览器控制台执行：

```javascript
window.smartColorManager.refresh();
```

## 技术细节

### Canvas 采样
- 使用 Canvas API 绘制背景图片
- 考虑 CSS `background-size: cover` 的缩放和裁剪效果
- 计算正确的图片绘制位置和尺寸

### 性能优化
- 使用采样间隔（每 4 个像素采样一次）
- 只分析顶部关键区域
- 跳过透明像素（alpha < 128）

### 跨域处理
- 设置 `crossOrigin = 'Anonymous'` 支持外部图片
- 对于无法跨域访问的图片，会降级到默认浅色文字

## 示例场景

### 场景 1：深色风景照
- 背景：夜景、深色森林等
- 结果：使用**白色文字** + 深色阴影

### 场景 2：浅色风景照
- 背景：雪景、沙滩、天空等
- 结果：使用**黑色文字** + 浅色阴影

### 场景 3：纯色背景
- 深色背景（如 #1a1a1a）：白色文字
- 浅色背景（如 #ecf0f1）：黑色文字

## 调试模式

### 开启调试模式
按 **Ctrl + Shift + D** 开启/关闭调试模式

### 调试模式功能
1. **可视化采样区域**：在屏幕上显示红色和绿色方框，标识系统正在分析的区域
   - 🔴 红色方框：日期文字采样区域
   - 🟢 绿色方框：星期文字采样区域

2. **详细控制台日志**：
   ```
   🖼️ 检测到背景图片: https://...
   🔍 采样区域 1: x=384, y=36, w=230, h=36
   📊 采样像素数: 5520, 平均亮度: 78.3%
   🔍 采样区域 2: x=691, y=36, w=153, h=36
   📊 采样像素数: 2760, 平均亮度: 81.2%
   💡 各区域亮度: [78.3%, 81.2%], 平均: 79.8%
   🎨 智能颜色调整: 背景亮度=79.8%, 模式=light, 文字=深色
   ```

3. **方框会在 3 秒后自动消失**

### 手动刷新
在浏览器控制台执行：

```javascript
// 刷新颜色分析
window.smartColorManager.refresh();

// 开启调试模式
window.smartColorManager.debugMode = true;
window.smartColorManager.refresh();
```

## 调整建议

如果你发现某些图片的颜色判断不准确，可以尝试：

1. **按 Ctrl+Shift+D 开启调试模式**，查看采样区域是否正确覆盖文字位置
2. **查看控制台日志**，了解实际检测的亮度值
3. **调整亮度阈值**：编辑 `smartColor.js` 中的 `threshold` 值（当前为 60）
4. **调整采样区域**：编辑 `smartColor.js` 中的 `sampleRegions` 数组

## 故障排除

### 问题：文字颜色没有变化
**解决方案**：
1. 检查浏览器控制台是否有错误
2. 确认 `smartColor.js` 已正确加载
3. 手动执行 `window.smartColorManager.refresh()`

### 问题：颜色判断不准确
**解决方案**：
1. 检查背景图片是否跨域
2. 调整亮度阈值（默认 50%）
3. 查看控制台输出的亮度值

### 问题：跨域图片无法分析
**解决方案**：
- 使用本地图片或支持 CORS 的图片服务
- 当前使用 Picsum 随机图片服务已支持 CORS
