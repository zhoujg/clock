# 音乐可视化功能

## 功能概述

现在粒子动画可以跟随背景音乐的节奏跳动，创造出动态的视觉效果！

## 实现原理

### 1. Web Audio API 音频分析
- 使用 `AudioContext` 和 `AnalyserNode` 实时分析音频频率数据
- 将音频分为低频（bass）、中频（mid）、高频（treble）三个频段
- 计算整体音量强度（overall）

### 2. 粒子动态响应
- **粒子大小**：根据音频强度动态调整粒子半径
  - 基础半径：2.5px
  - 音乐缩放范围：1.0 - 2.5 倍
  - 计算公式：`粒子大小 = 基础半径 × (1.0 + (整体音量 × 0.6 + 低频 × 0.4) × 1.5)`

- **粒子透明度**：音乐越强，粒子越亮
  - 基础透明度：0.6
  - 随音乐增加：最高可达 1.0

- **平滑过渡**：使用缓动动画，避免突兀的跳动
  - 缓动系数：0.3

### 3. 音频数据结构

```javascript
{
    bass: 0.0-1.0,      // 低频强度
    mid: 0.0-1.0,       // 中频强度
    treble: 0.0-1.0,    // 高频强度
    overall: 0.0-1.0,   // 整体音量
    raw: Uint8Array     // 原始频率数据
}
```

## 修改的文件

### 1. `scripts/bgmPlayer.js`
- 新增 `initAudioAnalyser()` 方法：初始化 Web Audio API 分析器
- 新增 `getAudioData()` 方法：获取实时音频频率数据
- 在音频播放时自动初始化分析器
- 支持 `.mpga` 格式音乐文件

### 2. `scripts/particle.js`
- 新增 `musicScale` 和 `musicScaleTarget` 属性：音乐缩放因子
- 修改 `update()` 方法：接收并处理音频数据
- 修改 `draw()` 方法：根据音乐强度动态调整粒子大小和透明度

### 3. `scripts/animation.js`
- 新增 `bgmPlayer` 属性：BGM 播放器引用
- 新增 `setBGMPlayer()` 方法：设置 BGM 播放器引用
- 修改 `animate()` 方法：获取音频数据并传递给粒子

### 4. `scripts/app.js`
- 在构造函数中调用 `animationManager.setBGMPlayer(bgmPlayerManager)`
- 建立动画管理器和 BGM 播放器之间的连接

## 使用方法

1. **开启粒子动画**：点击设置面板中的"动画线条"按钮
2. **播放音乐**：点击顶部音乐按钮，选择喜欢的音乐播放
3. **享受效果**：粒子会自动跟随音乐节奏跳动

## 技术细节

### 音频分析参数
- FFT 大小：256
- 频率分辨率：128 bins
- 低频范围：0 - 1/3 bins
- 中频范围：1/3 - 2/3 bins
- 高频范围：2/3 - 1 bins

### 视觉效果参数
- 低频权重：40%（提供强劲的脉冲感）
- 整体音量权重：60%（保持平滑的连续性）
- 最大缩放倍数：2.5 倍
- 缓动速度：30%

## 性能优化

- 只在音乐播放时进行频率分析
- 使用 `requestAnimationFrame` 保持流畅的 60fps
- 音频数据在 `animate()` 中统一获取，避免重复计算
- 平滑过渡避免 GPU 过度渲染

## 兼容性

- 支持所有现代浏览器（Chrome, Firefox, Safari, Edge）
- 需要 Web Audio API 支持
- 在不支持的浏览器中，粒子动画仍正常工作，只是不会响应音乐
