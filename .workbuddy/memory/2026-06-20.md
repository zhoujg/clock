
## 12:35 - 倒计时插件：横幅定位改为动态计算
- **问题**：倒计时横幅使用固定 `left:15%; top:calc(50%+4vw)` 定位，横屏时位置偏移严重
- **方案**：移除 CSS 固定定位，改为 JS `positionBanner()` 动态计算
- **定位逻辑**：查找 `.tick` 元素的第一个 `.tick-segment`（小时卡片），将横幅放在其左下方 12px 处
- **自适应**：监听 resize + orientationchange 事件自动重算位置；含边界保护（不超出屏幕、不遮挡工具栏）
- **文件**：`plugins/countdown/style.css`（移除 left/top/transform）、`plugins/countdown/countdown-class.js`（新增 positionBanner 方法）
- 版本号更新至 `v=20260620b`

## 12:53 - 倒计时横幅：修复横屏下垂直间距过大

- 版本号更新至 `v=20260620c`

## 13:18 - 倒计时横幅：再次修复横屏定位（选择器问题）
- **根本原因**：之前选择器 `:scope > .tick-segment` 无法匹配（中间隔了 `[data-layout]` div），最终回退到整个 `.tick` 容器，导致 `rect.bottom` 是全部3个卡片+名言区域的底部
- **修复**：重写选择器优先级链：
  1. `.tick-flip-panel` — 直接找第一个翻牌面板（最精确，这是用户实际看到的卡片）
  2. `.tick-segment` — 找第一个分段容器
  3. `[data-key="hours"]` — 原始 data-key 属性
  4. `.tick` — 兜底
- **间距进一步缩小**：横屏 GAP=6px, LEFT_OFFSET=0；竖屏 GAP=8px, LEFT_OFFSET=2
- 版本号更新至 `v=20260620d`

## 16:00 - 倒计时横幅：文字颜色智能调整 + 修复空引用错误
- **问题1**：横幅文字颜色硬编码，浅色背景看不清
- **问题2**：前一版只加了 `applyColors` 调用但没有定义方法，导致 `TypeError: this.adjustCountdownBannerColors is not a function`
- **修复**：在 `smartColor.js` 中新增 `adjustCountdownBannerColors()` 方法
  - 深色背景（background brightness < 60）→ 白色文字 + 半透明白色背景 + 白色边框
  - 浅色背景（background brightness ≥ 60）→ 深色文字 + 半透明深色背景 + 深色边框
  - 文字添加 textShadow 增强可读性
- 版本号：`smartColor.js` → `v=20260620a`；`countdown/plugin.js` → `v=20260620f`

## 16:08 - 倒计时横幅：修复智能颜色不生效（时序问题）
- **根因**：`smartColorManager` 初始化时（延迟500ms）横幅 `#countdownBanner` 尚未创建，`adjustCountdownBannerColors` 直接 `return`。之后 banner 创建时不再触发颜色更新
- **修复1**：`smartColor.js` — `applyColors` 中缓存 `_lastTextColor/_lastShadowColor/_lastBorderColor`，`adjustCountdownBannerColors` 无参调用时自动使用缓存值
- **修复2**：`countdown-class.js` — `createBanner()` 中在 `document.body.appendChild(this.banner)` 后主动调用 `window.smartColorManager.adjustCountdownBannerColors()`
- 版本号：`smartColor.js` → `v=20260620b`；`countdown/plugin.js` → `v=20260620g`
