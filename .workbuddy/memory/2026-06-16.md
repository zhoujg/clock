# 2026-06-16 工作日志

## 用户注册登录方式修改：邮箱 → 手机号

### 修改内容

#### 后端（PHP）
1. **`server/sql/schema.sql`**
   - `users` 表：`email VARCHAR(255)` → `phone VARCHAR(20)`
   - 唯一索引：`uk_email` → `uk_phone`

2. **`server/helpers.php`**
   - 删除 `isValidEmail()` 函数
   - 新增 `isValidPhone()` 函数：验证中国大陆和香港手机号（支持 `+86` 和 `+852` 前缀）

3. **`server/auth.php`**
   - `generateToken($userId, $email)` → `generateToken($userId, $phone)`
   - JWT payload：`email` → `phone`

4. **`server/api/register.php`**
   - 请求参数：`email` → `phone`
   - 验证逻辑：使用 `isValidPhone()` 验证手机号
   - 数据库操作：存储纯净手机号（去掉 `+86`/`+852` 前缀）
   - 返回数据：使用 `phone` 而不是 `email`

5. **`server/api/login.php`**
   - 请求参数：`email` → `phone`
   - 数据库查询：按 `phone` 查找用户
   - 返回数据：使用 `phone` 而不是 `email`

#### 前端（JS）
1. **`scripts/cloudSync.js`**
   - `get userEmail()` → `get userPhone()`
   - `register(email, password)` → `register(phone, password)`
   - `login(email, password)` → `login(phone, password)`

2. **`scripts/authUI.js`**
   - 入口按钮显示：使用 `userPhone` 而不是 `userEmail`
   - 登录/注册表单：邮箱输入框 → 手机号输入框（`type="tel"`，`maxlength="13"`）
   - 表单验证：增加手机号格式校验（支持中国大陆11位和香港8位）
   - 提交逻辑：传递 `phone` 参数给 API

#### Android 同步
- 已同步 `cloudSync.js`、`authUI.js` 和 `helpers.php` 到 `android/app/build/intermediates/assets/debug/public/`

### 手机号格式规则
- **中国大陆**：`13800138000` 或 `+8613800138000`（11位，以1开头）
- **中国香港**：`61234567` 或 `+85261234567`（8位，以5-9开头）
- 数据库统一存储为纯数字格式
- 前端验证：先去掉 `+86`/`+852` 前缀，再验证纯净手机号
