## MAMP MySQL 连接修复

### 问题
MAMP 的 socket 文件 `/Applications/MAMP/tmp/mysql/mysql.sock` 在磁盘上存在，但 PDO 通过 Unix socket 连接时报 `SQLSTATE[HY000] [2002] No such file or directory`。数据库名已手动改为 `clock`。

### 修复
`server/db.php`：MAMP 环境不再使用 Unix socket（`mysql:unix_socket=...`），改为 TCP 连接 `127.0.0.1:8889`，用户名/密码为 `root`/`root`。这样完全避免了 socket 文件权限或路径兼容性问题。

### 数据库初始化
通过 PHP PDO 直连创建了 `clock` 数据库和三张表（users/user_data/stories），并插入了默认用户 `44837939`。

## 登录后设置被错误关闭的 Bug 修复

### 根因
`app.js` 的 `loadSavedSettings()` 中通过 `if (settings.xxx) { manager.toggle() }` 来恢复状态，但 `toggle()` 是**翻转**操作（`enabled = !enabled`），而不是设置为指定值。

页面初次加载时 `enabled = false`，`toggle()` 正确开启。但登录后再次调用 `loadSavedSettings()` 时 `enabled` 已经是 `true`，再 `toggle()` 反而把它关掉了。

### 修复
`app.js` 两处改为比较后再决定：
```js
if (settings.animationEnabled !== this.animationManager.enabled) { this.animationManager.toggle(); }
if (settings.tickSoundEnabled !== this.tickSoundManager.enabled) { this.tickSoundManager.toggle(); }
```

### 第二次修复（setEnabled）
toggle() 比较方式仍不可靠，改为给两个 manager 新增 `setEnabled(enabled)` 方法：
- `tickSound.js`：新增 `setEnabled()`，`toggle()` 改为调用 `setEnabled(!this.enabled)`
- `animation.js`：新增 `setEnabled()`，`toggle()` 改为调用 `setEnabled(!this.enabled)`
- `app.js`：`loadSavedSettings()` 直接调用 `setEnabled()`

## App 天气显示旧数据

### 问题
手机浏览器访问 neihou.cn/clock/ 天气正确（下雨），但 App 中显示晴天（旧数据）。App 加载的是同一远程 URL（Capacitor server.url），但 WebView 有独立的 localStorage，天气缓存 30 分钟不刷新。

### 修复
`scripts/weather.js`：
1. `loadWeatherData()` 改为：先展示缓存 → 后台异步拉取最新数据 → 静默更新
2. API 失败时优先使用最近缓存，不再直接跳到随机模拟数据

### WebView 缓存问题深入分析
- 根本原因：App WebView 的 HTTP 缓存比手机浏览器更激进，缓存了旧版 JS/CSS 文件
- `index.html` 所有 JS/CSS 引用加了 `?v=20260616` 版本参数，强制 WebView 重新加载
- Nginx 建议配置：`index.html` 设 `Cache-Control: no-cache`，确保 HTML 不被缓存
