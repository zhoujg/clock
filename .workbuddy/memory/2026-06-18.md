## 收藏图片重叠修复（多次迭代）
- 问题：竖屏手机上随机图片背景的收藏列表中图片重叠
- 根因：部分手机浏览器不支持 `aspect-ratio`，导致 `.favorite-item` 高度为 0
- 修复历程：
  1. 改用 `aspect-ratio` + `min-height: 80px` → 部分手机仍不支持
  2. 改用 `::before` 伪元素 + `padding-top: 75%` → `.favorite-preview` 的 `height:100%` 在某些手机上解析为 0
  3. 最终方案：`.favorite-item` 用 `height:0; padding-top:75%`（经典技巧），`.favorite-preview` 用 `top:0; left:0; right:0; bottom:0` 四边定位填充
- 涉及文件：`styles/picsum.css`

- Fallback：未配置 API Key 时自动使用本地模拟数据，不影响正常使用

## AI 内容模块 URL 修复（2026-06-18 晚间）
- 问题：`[AIContent] 初始化语录失败: TypeError: Failed to construct 'URL': Invalid URL`
- 根因1：`API_BASE` 使用了 `window.location.origin + '/api'`，与 `cloudSync.js` 的路径不一致（cloudSync 用 `/server/api` 或 `/clockserver/api`）
- 根因2：`new URL('/api/ai-content.php')` 单参数要求绝对 URL，相对路径会报错
- 修复：
  - `API_BASE` 改为与 `cloudSync.js` 一致：本地 `/server/api`，线上 `/clockserver/api`
  - 弃用 `new URL()`，改用 `new URLSearchParams()` + 字符串拼接相对路径（与 cloudSync.js 一致）
  - 修复 `fetch` headers 合并 bug（展开对象会覆盖而非合并 headers）
  - `index.html` 版本号更新为 `v=20260618b`

## 天气解读样式协调修复（2026-06-18 深夜）
- 用户反馈：AI 天气解读文本显示不协调，与日期行混在一起缺乏层次感
- 问题分析：
  - `white-space: nowrap` 导致长文本不换行，挤成一行
  - `left: 50% + translateX(-50%)` 居中定位，与手机竖屏日期行左对齐不一致
  - 间距仅 `4px`，和日期行几乎贴在一起
  - 背景透明度不够，看起来像日期行的延续
- 修改（`styles/ai-content.css`）：
  - 去掉 `white-space: nowrap` → 改为 `white-space: normal` + `word-break: break-word`
  - 定位从居中 `left:50%; transform:translateX(-50%)` 改为左对齐 `left:0`（与竖屏日期行一致）
  - 间距从 `4px` 增加到 `10px`（桌面）/ `8px`（手机）
  - 背景从 `rgba(0,0,0,0.12)` 加深为 `rgba(0,0,0,0.18)`，毛玻璃从 `blur(6px)` 增强为 `blur(10px) saturate(120%))`
  - 圆角从 `10px` 增大到 `12px`，行高从 `1.5` 增加到 `1.65`，字间距 `letter-spacing: 0.5px`
  - 对齐改为 `text-align: left`，与左对齐的日期行保持视觉一致性
  - 手机竖屏字号从 `3.2vw` 调整为 `3vw`，更协调
- 版本号：CSS → `v=20260618e`
